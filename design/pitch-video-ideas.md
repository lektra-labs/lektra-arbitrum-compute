# Lektra Pitch Video Ideas

## What Lektra Is
Lektra is a distributed AI compute platform. A user submits one prompt, Lektra sends the work across multiple compute nodes, returns generated outputs, and keeps a verifiable payment trail for what succeeded, what failed, and what should be paid.

## Idea 1: Creative User Story
Best for: simple hackathon pitch, non-technical audience

### Core message
Lektra makes AI generation scalable and fair behind the scenes.

### Flow
1. Start with the problem.
   AI apps need a lot of compute, and single-provider systems are expensive and fragile.
2. Introduce Lektra.
   Lektra is a distributed AI compute network for creative generation.
3. Show the product.
   A user types a prompt and 4 images appear.
   Narration: One prompt goes in. Multiple results come back.
4. Reveal the hidden value.
   Open the execution timeline.
   Show that 3 nodes were used.
   Show one node failed and another recovered the task.
   Narration: Lektra keeps the experience smooth even when a machine fails.
5. End with trust.
   Open the settlement proof.
   Show failed attempt = disputed, successful recovery = successful.
   Narration: Lektra does not just run the job. It records exactly what happened so payment follows reality.

### Ending line
Lektra turns distributed AI compute into a product creators can use without worrying about the infrastructure underneath.

### Why this works
- Easy to understand
- Starts with images, not blockchain
- Good for judges who care about product clarity

## Idea 2: Infrastructure Trust Story
Best for: technical judges, infra/blockchain audience

### Core message
Lektra combines distributed AI execution with verifiable settlement.

### Flow
1. Start with what Lektra is.
   Lektra is a distributed AI compute platform that routes one task across multiple nodes and settles outcomes transparently.
2. Show the challenge.
   In distributed systems, not every execution succeeds. Nodes fail. Jobs get rerouted. Payments can become ambiguous.
3. Show Lektra's answer.
   Display prompt to execution timeline.
   Highlight: `Node 003 - Exxact -> Node 001 - Lektra`
   Narration: This task started on one node, failed, and was recovered by another.
4. Show the settlement model.
   Click the red failed bar.
   Show disputed settlement proof.
   Click the green recovery bar.
   Show successful settlement proof.
   Narration: Lektra treats these as separate outcomes. Failed work is disputed. Recovered work is settled successfully.
5. End with platform value.
   Show billing trace.
   Narration: That gives operators, customers, and the network a shared source of truth for execution and payment.

### Ending line
Lektra makes AI compute not just distributed, but accountable.

### Why this works
- Stronger technical differentiation
- Highlights failure recovery and settlement logic
- Better if judges care about architecture and trust

## Recommendation
Use Idea 1 if the audience is mixed or non-technical.

Use Idea 2 if the audience is more technical and you want to emphasize the settlement layer.
